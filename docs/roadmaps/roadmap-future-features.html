Sprint J — Jarvis Intelligence Upgrades (Complete — 2026-04-25)

Voice quality, smart context injection, and conversational polish for Jarvis
J1 ✓
Voice Upgrade — Puter.js + ElevenLabs
Replaces Web Speech API with Puter.js + ElevenLabs Daniel voice (British, authoritative). Fallback chain to Web Speech API on failure. Voice on/off toggle (🔊/🔇) in Jarvis panel header.
J3 ✓
Smart Context Injection (Tiered)
selectRelevantCards() injects up to 20 tiered cards into every Jarvis message. Tier 1: named cards (full). Tier 2: matched-category cards ranked by recency + word count + connections (full, ≤7). Tier 3: additional matches truncated to 200 chars (≤10). Smart caching reuses selection when topic hasn't shifted (≥50% category overlap). Replaces first-turn-only buildStoryContext().
J5 ✓
Follow-Up Questions When Ambiguous
Jarvis asks one focused follow-up when a request is ambiguous. Biases toward stating assumptions over asking. Voice mode stricter threshold (answer >3 sentences + generative task + interpretations share <30%). User override: "just answer" or "continue" skips asking.
J2 ✓
Mic Button Guard
Chat mic button now hidden entirely on browsers without SpeechRecognition support (Firefox, Safari). Chrome/Edge show the button normally.

UX Sprint G — Map Organization Zones (Later)

Brings semantic spatial structure to the free-form map — optional, non-destructive
Structured Map Layout with Zones
G1
Medium Priority No dependencies
Done when: "Organize by Zone" snaps all cards into semantic regions: arcs across the top, characters below arcs, world-building on the right, quotes & ideas at the bottom
  • "Organize by Zone" button in the map toolbar (next to existing "Re-organize")
  • Arc zone: arc cards laid out horizontally across the top row (~y=80), evenly spaced in current story order
  • Character zone: character cards arranged below the arc they're most connected to (fallback: general character row at y=320)
  • World zone: world-building cards clustered in a column on the right side of the map
  • Quotes & Ideas zone: quote and idea cards arranged in a horizontal strip at the bottom (~y=560)
  • Layout is computed once; positions saved to sf_positions — free-form dragging still works after
  • Does not auto-run — user must explicitly trigger it; existing positions preserved until then
Arc-to-character zone placement uses existing sf_connections data to determine relatedness
Zone Background Labels
G2
Low Priority
Done when: The map canvas shows faint labeled background regions for each semantic zone, making the structure readable at a glance
  • SVG background regions: translucent rounded rectangles drawn behind card content
  • Each region labeled in its top-left corner with a small, dim zone name ("Arcs & Plot", "Characters", "World Building", "Quotes & Ideas")
  • Regions are fixed to the canvas coordinate system (not the viewport) so they stay with the cards as you pan/zoom
  • "Show Zones" toggle in map toolbar; state persisted in sf_map_zones_visible
Requires G1 (Structured Layout) to determine zone positions
