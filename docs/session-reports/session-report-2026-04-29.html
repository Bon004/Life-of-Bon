Decisions made:

  • OKLCH over hex/rgba for all tokens. Perceptually uniform color space; chroma automatically reduces near lightness extremes, preventing garish values. All CSS variables in :root rewritten.
  • color-mix() for card tints. Category tints expressed as color-mix(in srgb, rgb(var(--cat-char)) 7%, var(--surface)) — keeps category color meaningful without the old left-border stripe. Requires RGB triplet format for the custom properties.
  • Uniform spherical distribution for orb particles. Previous ring-based layout created visible latitude bands that looked solid at small canvas sizes. Math.acos(1 - 2 * random()) for phi gives true uniform surface distribution.
  • Deleted redesign-preview.html. All validated designs are now in production. The file had no ongoing value and was adding noise to git status.
  • border-top instead of border-left for home nav tiles. The D1 block used border-left stripes as a color accent — banned pattern. border-top gives category color meaning (a cap/header accent) without the stripe cliche.
