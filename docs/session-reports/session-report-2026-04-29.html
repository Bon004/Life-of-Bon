Decisions made:

  • Format-based column labels, not genre-based. "Genre" is a distinct concept reserved for future filtering. Format (novel, biography, memoir, nonfiction, screenplay, other) drives the default column vocabulary. Perplexity validated this distinction.
  • Explicit strip abbreviations in FORMAT_COLUMNS. Each column entry has both label (full) and strip (explicit 4-char abbreviation), not derived via label.slice(0,4). Perplexity recommended explicit abbreviations.
  • projectFacts / assistantBehavior separation in Sage prompt. Project metadata and tone behavior are independent blocks. Perplexity recommended this separation.
  • Use .hidden CSS class (not hidden attribute) for any modal that lives inside an element with a display CSS rule. The HTML hidden attribute loses to author-stylesheet display:flex. .hidden { display: none !important; } always wins. This is now the established pattern for all modals.
  • Node stdin parse check as first-line syntax validator. node --input-type=module < app.js catches syntax errors instantly and reports the exact line. Run this after any AI-generated string literal code before committing.
