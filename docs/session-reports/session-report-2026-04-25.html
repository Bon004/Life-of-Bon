07Open Questions & Gaps

  • ElevenLabs Daniel voice may or may not play via Puter.js — Puter.js is a third-party service that wraps ElevenLabs. The voice name "Daniel" is passed to Puter.js. If Puter.js doesn't resolve that name correctly, it will fail silently and fall back to Web Speech API (Aria Natural). Check the browser console for [Jarvis TTS] Puter.js/ElevenLabs failed to know which path is running.
  • Puter.js requires a free account login — Puter.js may prompt the user to sign in to puter.com on first TTS call. This is part of their free-tier model. It's worth confirming this doesn't create friction in normal use.
  • Voice ID vs. voice name — ElevenLabs uses internal voice IDs, not names. Puter.js may accept the string "Daniel" directly, or it may require the ElevenLabs voice ID. If Daniel isn't working, the next step is to find the correct ID from Puter.js docs or the ElevenLabs voice library.
  • J3 context injection: token cost — Smart context injection now runs every Jarvis turn (not just the first). The smart cache mitigates most of the cost on multi-turn threads, but first-turn and topic-switch turns will inject up to 20 cards. For 200+ cards and typical content lengths, this could add 1,000–3,000 tokens per cold-start turn. At personal use scale (cents/day) this is fine, but worth monitoring if conversation volume is high.
  • Roadmap is 4 local commits ahead of GitHub — commits d034e8f, 3ebbc04 (baseline), 54e1b10, e6d9c92 are local only. User has not authorized a push yet.
