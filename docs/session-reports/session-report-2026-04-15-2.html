Interaction Style

You work in short, iterative bursts of visual feedback, especially on styling. Rather than writing detailed specs upfront, you give Claude directional prompts like "make it bolder" or "swap to teal/blue" and then react to what lands. This shows up clearly in sessions where you asked for a redesign, saw the result was mostly color tweaks, and immediately pivoted to request layout changes and another palette shift.

You let Claude drive long procedural tasks without interrupting, sometimes to your detriment. The 90-minute Node.js setup saga on a locked-down work machine is the clearest example: you followed along through PATH issues, execution policy errors, and paths-with-spaces problems before finally calling it.

Your instructions tend to be terse and outcome-focused rather than prescriptive, which works great when Claude guesses right (violet/navy palette, committed cleanly) but leaves room for misfires when it doesn't. Think of it as an art-director style: you point at a direction and expect Claude to fill in the details correctly.

Hooks — auto-format after CSS edits

Auto-run shell commands at lifecycle events. With 26+ CSS edits per session, a PostToolUse hook running prettier would keep styles clean automatically.

// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",

    }]
  }
}

Usage Pattern Tips

1
Specify redesign scope upfront
Distinguish "palette swap" from "layout redesign" explicitly in your first message. Stating scope (layout, spacing, typography, components) prevents a wasted pass and a second frustrated request.
Copyable prompt
I want a BOLD redesign of the homepage — this means layout, spacing, typography, and component structure, NOT just colors. Keep the current violet/navy palette. Show me a plan before editing.
2
Bail out of environment rabbit holes faster
Set a time/attempt budget when setting up tooling on restricted machines. Tell Claude upfront about admin/PATH constraints and cap attempts, so it pivots to portable Node, Docker, or a cloud IDE early.
Copyable prompt
I'm on a locked-down Windows work machine, no admin, paths have spaces. If standard Node install fails once, immediately pivot to a portable/no-install option or tell me to use a different machine.
3
Retry strategy for API 500s
When API errors hit mid-task, resume with explicit state so Claude doesn't re-plan from scratch. Restate the exact target state when picking back up.
Copyable prompt
Resuming after API error. Current state: layout redesign landed, palette still orange/black. Target: teal/blue accents across all CSS files. List files you'll touch, then apply.
