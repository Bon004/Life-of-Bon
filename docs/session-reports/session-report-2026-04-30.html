  • style.css — .chat-autosend-toggle styles (active/inactive states, hover); .chat-pause-sage styles (:disabled dimmed, .paused accent pulse animation).
  • CODEBASE.md — sf_voice_autosend added to global localStorage key table.

Decisions made:

  • Debounce via continuous=true + 2500ms timer instead of longer silence threshold. continuous = false gives no control over the browser's silence detector. continuous = true + a debounce timer on every result event is the only reliable cross-browser pattern. 2500ms was validated by Perplexity as long enough for natural pauses without feeling sluggish.
  • userStoppedMic flag on recognition object (_userStopped property). The mic stop branch lives outside the recognition closure. The Quest Giver stored the flag as a property on the shared orbRecognition reference and reads it in onend. Functionally equivalent to the plan's closure variable.
  • shouldAutoSend snapshotted inside the timer callback, not at onresult time. Prevents a race where the user toggles the flag mid-session. The behavior that was active when the timer fires is the one that applies — not what was active when the user started speaking.
  • onend orphan guard sends only on unexpected termination. userStoppedMic = true signals a deliberate user stop — partial text is discarded, not sent. Network drops and browser timeouts leave userStoppedMic = false, so accumulated text is sent (respecting autoSendEnabled).
  • HTML Audio uses .pause()/.play() for pause/resume (not .resume()). .resume() does not exist on HTML Audio elements. audio.pause() preserves currentTime; audio.play() resumes from that position. Perplexity flagged this explicitly — it's a silent TypeError if called.
  • processTTSQueue Guard 0 auto-clears stale ttsPaused. Prevents queue collision: a new reply arriving while Sage is paused would be held hostage by the old pause flag. Guard 0 resets it when starting a fresh queue with nothing playing.
  • Three-party plan review before any code was written. Two full Perplexity passes on the plan file (not just prose summaries) before delegation. Perplexity's third review caught that their own second-review had been addressed but the prose summary didn't match the code blocks — confirmed by reading the actual file line numbers.
