Decisions made:

  • Global rotation derived from orbConstellationT × 0.04, not a separate clock. Tying the angle to the existing time variable means rotation speed automatically tracks state (idle: ~0.024 rad/sec; thinking: ~0.101 rad/sec) and freezes on paused — zero extra conditionals or globals. Perplexity confirmed this as the clean architectural choice.
  • Store rotated coords as p.rx/p.ry, never modify p.bx/p.by. Base positions are the permanent anchors for future frame rotations. Mutating them would cause drift accumulation over time. The rotated variants are ephemeral per-frame values.
  • Paragraph-aware excerpt over character slice. Perplexity explicitly flagged that text.slice(-1600) cuts through sentences and creates misleading context. Last-2-paragraph extraction respects semantic boundaries and is more faithful to what the user is actually reading. The excerpt is shorter on average (a paragraph is rarely 800 chars) so token cost is lower too.
  • Editor context injected before chat_memory. Live DOM text must outrank auto-summarized old chat messages. If both appear in the system prompt, Claude gives more weight to earlier context. Placing the live editor snippet first ensures it wins when the two conflict.
  • Two-layer editor access: always-on snapshot + on-demand tool. The always-on snapshot (last 2 paragraphs) handles the most common question ("what's the last paragraph?") with zero round-trips. The read_editor tool handles full reads and section-specific queries without inflating every prompt with thousands of words.
  • Three-party plan review for both tracks before any code. Perplexity reviewed both Plan A and Plan B, adding the paragraph-aware refinement to B and confirming Plan A's global rotation architecture. Both plans were tightened before delegation.
