Decisions made:

  • Ask Claude moved to header, not kept as FAB. The floating button was covering the summary bar's Refresh/▲ buttons even after being raised to 96px in Run 2. Moving it into the header gives it a permanent, predictable position that doesn't overlap anything. The button is sized to match the other header buttons. Rejected: keeping as FAB with higher z-index or different corner (left/top) — all FAB positions create overlap risk as the app grows.
  • Timeline resize clamped to 80–600px. 80px is the minimum to show at least one row of arc chips without cutting them off. 600px is the maximum before the brainstorm section becomes too cramped on a standard laptop screen (1080p). Rejected: unclamped resize (allows the timeline to fill the entire panel, hiding brainstorm content).
  • F&R replace regex changed to 'gi' (global, case-insensitive). The previous 'i' flag only replaced the first occurrence regardless of which match was highlighted. Using global + tracking the target index via frCurrentMatch allows precise per-match replacement. After replacing, frCurrentMatch is decremented by 1 before calling frNavigateMatch(1) so the cursor naturally advances to the next match. Rejected: DOM range-based replacement (complex, fragile with contenteditable).
