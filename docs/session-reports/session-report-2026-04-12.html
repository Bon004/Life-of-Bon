What We Built

1
FAB Button Overlap Fix

The "Ask Claude" floating action button was covering the last lines of scrollable content in every tab.

Fix
  • Added padding-bottom: 80px to .col-cards, .writing-editor, .char-profile-panel, .char-list, and .brainstorm-panel
  • Merged into shorthand where existing padding was already set (e.g., padding: 24px 32px 80px)
2
Archive Left Slide-in Panel

Archived cards used to remain visible in board and map views with a faded style. They are now fully hidden and browseable in a dedicated left panel.

Behaviour Changes
  • renderCards() and renderMap() now filter out cards where status === 'archived'
  • Left slide-in panel (260px) lists archived cards grouped by type with a Restore button on each
  • Panel toggled via "📦 Archive" button in the header; overlay click closes it
  • renderArchivePanel() called whenever cards are archived or restored
Design
  • transform: translateX(-100%) / .open { transform: translateX(0) } slide pattern (mirrors right chat panel)
  • Cards shown with type-colored left border and compact preview text
3
Home Page Tab

A proper landing screen shown on app load — replacing the previous behavior of dropping directly into Story Canvas.

Contents
  • Hero: "StoryForge" title with gradient text (orange → white), subtitle "Your world. Your arcs. Your story."
  • Stats chips: live card count, arc count, character count (glass-morphism style)
  • Prominent CTA: "Get Started →" button navigates to Story Canvas (primary workspace)
  • Nav tiles: 4 secondary buttons for Story Canvas, Writing, Characters, Arcs
  • Recent cards strip: last 5 cards added as mini card previews
Implementation
  • Home tab is first and active by default on load
  • renderHomePage() called on load and after card changes to keep stats/recent strip live
4
Writing Tab — Rich Text Editor

The Working Copy textarea was replaced with a contenteditable div and a formatting toolbar.

Toolbar Buttons
  • B (bold), I (italic), U (underline), H1, H2, bullet list, numbered list, HR divider, clear formatting
  • Uses document.execCommand() — intentional choice over Selection API for simplicity in plain-JS app
  • mousedown + e.preventDefault() preserves editor focus before command runs
Data Handling
  • Saved as innerHTML to sf_writing_copy
  • AI actions receive textContent (plain text — no HTML noise in prompts)
  • Word count reads .textContent not .value
5
Character Grouping by Name

Multiple cards for the same character (e.g., "Bon", "Bon - Child", "Young Bon") are now grouped in the Characters sidebar.

Grouping Logic
  • groupCharactersByName(): normalises titles, extracts first meaningful token as root name, groups cards that share a root
  • Groups with 2+ cards render as an expandable header with count badge
  • Single cards render as before
Profile View
  • When a group is selected, the profile panel shows the shared profile form plus a "Related Cards" section listing all cards in the group
6

Several interaction improvements to the Arcs & Timeline tab.

Arc Timeline

Decisions Made

DecisionRationale
Left slide-in panel for archiveMirrors existing chat panel pattern; keeps UI consistent without introducing a new design paradigm
document.execCommand() for rich textDeprecated but fully functional; Selection API alternative would add 200+ lines for no user-visible gain in a plain-JS personal app
Save writing as innerHTML, send to AI as textContentPreserves formatting locally; AI prompts stay clean with no HTML noise
Root-name grouping for charactersFirst meaningful token handles "Bon", "Bon - Child", "Young Bon" without requiring the user to tag or link cards manually
▲▼ buttons instead of drag for Situations reorderSimpler to implement and verify; drag-and-drop on a 36-item grid is error-prone on desktop and unusable on touch without significant extra code
/insights produces two files, not oneSession report = focused snapshot; full-project report = always up-to-date reference. Mixing them means neither is useful for its purpose.

Known Risks & Watch Points

execCommand deprecation — Browsers still support it fully, but if a future Chromium update removes it, the rich text toolbar will silently stop working. No short-term risk; would need Selection API rewrite if it breaks.
Writing content migration — sf_writing_copy now stores HTML. If a user has plain text saved from before this session, it will load into the rich editor as plain text — which is harmless, but H1/H2/list formatting won't be present. No data loss risk.
Character group root-name collisions — Two unrelated characters whose names start with the same token (e.g., "Bon" the protagonist and a background character called "Bonwick") would be incorrectly grouped. Acceptable for now; can be refined with explicit "link cards" feature later.
Archive panel z-index on mobile — The archive panel is z-index: 200 and the chat panel is similar. On narrow screens both could overlap if opened simultaneously. No toggle-both guard exists yet.
