Usage Pattern Tips

1
Too many empty /compact sessions
Half your sessions produced nothing because they were continuations that immediately compacted or hit usage limits. Only compact when context is genuinely full, and start new sessions fresh for unrelated tasks instead of chaining via /compact.
  • Before compacting, summarize what's still pending to decide whether to continue or start fresh
  • Batch related questions into a single session rather than spreading across many
2
Trust user bug reports upfront
In your broken-buttons session, Claude wasted turns verifying the bug existed. Phrasing like "X is broken — find why" sets the right frame immediately and skips the verification dance.
  • Use: "The buttons on index.html don't respond to clicks at all. Find the root cause — don't verify they're broken, I've confirmed it."
  • Include: what IS working alongside what ISN'T, to narrow the search space
3
Batch your StoryForge questions
Your most successful session tackled 4 related tasks together (run app, PDF upload, duplicate cleanup, Git). Bundling related questions gives Claude project context and avoids re-exploration across sessions.
  • Use: "I need help with StoryForge: (1) run it locally, (2) add feature X, (3) commit changes. Plan these in order."
  • One focused session beats three fragmented ones for the same total work

On the Horizon

Deploy an agent that auto-detects your shell environment (PowerShell vs bash) and rewrites commands accordingly, preventing the syntax errors you hit. Could maintain a project runbook that evolves as it learns your stack.
