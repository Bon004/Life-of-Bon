What We Built

1
Writing Tab

A Word-style writing environment with two clearly separated panes — one for the official story, one for AI-assisted exploration.

Panes & Layout
  • Working Copy — the canonical, official version of the story (left, 60%)
  • Draft pane — AI-assisted scratchpad toggled by button (right, 40%)
  • "← Promote to Working Copy" button copies draft content over
Features
  • Live word count on both panes
  • Auto-save to localStorage (sf_writing_copy, sf_writing_draft)
AI Actions (dropdown)
  • Generate Draft from Notes — writes ~600–1200-word narrative from story cards → draft pane
  • Continue Writing — appends ~300 words from current content → draft pane
  • Improve This — rewrites working copy for clarity & pacing → draft pane
  • Check Story Consistency — flags contradictions with cards → collapsible result panel
2
Characters Tab

A profile workspace that pulls character cards from the Story Canvas and lets you build a full profile for each one.

Layout
  • Sidebar (left) — scrollable list of all character-type cards; click to select
  • Profile panel (right) — full profile sheet for the selected character
Profile Fields
  • Role (dropdown: Protagonist / Antagonist / Supporting / Mentor / Unknown)
  • Enneagram Type 1–9 — auto-displays description card below (Core Desire, Core Fear, Key Traits)
  • Core Goal, Core Fear, Character Arc, Notes — all textareas
  • All fields auto-save on change to sf_character_profiles
AI Generate Profile
  • Fills only empty fields — never overwrites existing user content
3
Arcs & Timeline Tab

A brainstorming and planning workspace for arc cards, dramatic structure, and AI-powered plot ideas.

Timeline Strip (top)
  • Horizontal scrollable row of arc cards as clickable nodes
  • Empty state shown when no arc cards exist
Brainstorm Panel (middle)
  • 36 Dramatic Situations — 6-column grid; click to select + see description; feeds into Ideas Generator
  • 8-Sequence Structure — all 8 beats with descriptions + mapped arc cards; "Map My Arcs" AI button assigns arcs to slots
AI Ideas Generator (bottom)
  • Optional direction textarea + selected situation context
  • "Generate 3 Ideas ▶" → returns 3 numbered "What if..." plot directions

Decisions Made

DecisionRationale
Plan-first approachUser explicitly requested roadmap review before implementation began
Static data embedded in app.jsCLAUDE.md prohibits new files; no framework needed for constant arrays
Idempotent render guards on Situations/Sequences gridsPrevents duplicate rendering when switching between brainstorm sub-tabs
AI Generate Profile fills empty fields onlyRespects existing user work — never overwrites what the user has written
Working Copy / Draft distinctionKeeps the "official" writing separate from AI-assisted exploration
Event delegation on #panel-arcsBrainstorm tab buttons may not be in DOM until arcs panel is first activated

Known Risks & Watch Points

Tab hook fragility — renderCharacterSidebar() and renderArcsTab() are called by hooks appended to existing .tab-btn click listeners. If tab-switching code is refactored, these hooks need to be reviewed.
AI profile JSON parsing — generateCharacterProfile parses a JSON response from Claude. If the model returns non-JSON, there is a try/catch, but the user will see a toast error rather than a graceful fallback.
Situations grid on iPad — The 36 Situations grid uses grid-template-columns: repeat(6, 1fr). This will need responsive breakpoints for iPad (scheduled for Phase 4c).
