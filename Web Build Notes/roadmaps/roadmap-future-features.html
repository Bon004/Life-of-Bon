UX Sprint A — Card Polish & Import Staging (Do Now)

Highest impact · Foundational for all later UX work · Build in order: A1 → A2 → A3 → A4
Card Visual Refresh
A1
High Priority No dependencies
Done when: Cards look cleaner with better spacing, type badges, subtle color tints, and relative timestamps
  • Increase card padding (10→14px), gap between cards (7→10px), font sizes (+1px)
  • Add faint type-color background gradient tint (4% opacity)
  • Add small type pill badge (e.g., "character") on each card
  • Add relative timestamp below content ("2d ago", "1w ago")
  • Move delete button into hover/actions area for cleaner default state
Collapsible Card Content
A2
High Priority No dependencies
Done when: Long cards show first 2 lines with fade-out; click to expand/collapse
  • Cards >100 chars default to collapsed (max-height 3.2em with gradient fade)
  • Click card body to toggle expand/collapse
  • Auto-expand when user clicks to edit (contenteditable focus)
Quick-Add Card (No AI)
A3
High Priority No dependencies
Done when: User can add a card with title + content via inline form, no AI required
  • Replace prompt() dialog with inline form at top of board column
  • Form has: title input, content textarea, Add/Cancel buttons
  • Type pre-filled from the column; Enter to submit, Escape to cancel
Staged Import Preview
A4
High Priority
Done when: "Organize with AI" shows a preview with summary + card checklist instead of creating cards immediately
  • After AI returns, show preview screen inside modal (not instant card creation)
  • Preview shows: 2-3 sentence summary + proposed cards grouped by type
  • Each proposed card has a checkbox, inline-editable title and content
  • "Create Selected" button confirms; only checked cards are created
  • New card field: batchId linking cards from same import
  • New localStorage key: sf_batches with { summary, createdAt, cardCount } per batch
Core fix for the "too many cards" problem — enables batch grouping in Sprint B

UX Sprint B — Organization & Smarter AI (Do Next)

Builds on Sprint A · Better card grouping, character tags, archive upgrades
Smarter AI Prompt (Fewer, Richer Cards)
B1
High Priority
Done when: AI creates 3-7 rich cards (3-5 sentences each) instead of 10-15 thin ones
  • Change card content target from "1-3 sentences" to "3-5 sentences with full context"
  • Add consolidation instruction: merge fragments about same character/location
  • Change cap from 15 to 10 max, target 3-7
  • Update both buildPrompt() and buildSyncPrompt()
Batch Group View on Board
B2
Medium Priority
Done when: Cards from the same import are visually grouped with a collapsible batch header showing the import summary
  • Cards sharing a batchId render together with a batch header above them
  • Batch header shows: summary text, card count, collapse/expand toggle
  • Collapsing a batch hides its cards, showing only the summary line
  • Collapse state persisted in sf_collapsed_batches
Requires A4 (Staged Import) for batchId data
Character Tags & Role Filtering
B3
Medium Priority
Done when: Characters can be tagged (villain, mentor, royalty, etc.) and filtered by tag in the sidebar
  • Add tags: string[] to character profiles
  • Tag input with predefined suggestions in Characters tab profile panel
  • Tags render as colored pills on character cards everywhere
  • Tag filter bar at top of character sidebar
Archive with AI Summary
B4
Medium Priority
Done when: Archiving a card can optionally generate a 1-sentence summary card that stays visible
  • "Summarize before archiving?" prompt when archiving a card
  • Summary card stays active, marked with isSummary: true, styled differently (italic, dimmer)
  • Multi-select on board (Shift+click) → "Archive Selected" with batch summarize option

Recommended Build Order

  1. 1
    UX Sprint A (A1–A4) — Card visual refresh, collapsible content, quick-add, staged import preview. Fixes the card flood problem and makes everything scannable. This session.
  2. 2
    UX Sprint B (B1–B4) — Smarter AI prompt (fewer cards), batch group view, character tags, archive with summaries. Builds on Sprint A's batchId system.
  3. 3
    Phase 4b — Export / Import data. Download sf_cards as JSON; re-import with merge or replace. Data safety.
  4. 4
    UX Sprint C (C1–C3) — Import batch history, card sections/sub-cards, AI stale detection. Advanced organization once the basics are solid.
  5. 5
    Choose: Phase 5 (Writing depth) or Phase 6 (Characters depth — relationship web, tags) based on current story priorities.
  6. 6
    Phase 4c — iPad / mobile layout. Phase 7b — Chapter/scene breakdown. Phase 8 — Settings panel.

Intentionally Out of Scope

  • Server-side storage — stays localStorage by design
  • User accounts or multi-user collaboration
  • Build tools, bundlers, or npm packages — plain HTML/JS only per project rules
  • Real-time sync across devices
